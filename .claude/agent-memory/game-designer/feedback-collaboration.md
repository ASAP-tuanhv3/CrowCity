name: Collaboration style and feedback
description: How the user prefers to collaborate on design work — approval patterns, doc norms, tone
type: feedback

User provides detailed, precise design briefs with locked upstream contracts already resolved. They expect the game designer to synthesize those contracts into clean, implementable rule text — not re-open settled decisions.

**Why:** Sessions have included thorough upstream context (CSM, Relic, CCR GDDs all read before drafting). User values precision and coverage over exploration at the drafting stage.

**How to apply:** When given a brief with "framework decisions already locked — do not revisit," treat those as hard constraints. Produce ready-to-drop markdown. Do not propose alternatives to locked decisions. Do check cross-GDD consistency for contract details (e.g., tick ordering, API names, state names) to ensure the output matches what upstream docs specify.
