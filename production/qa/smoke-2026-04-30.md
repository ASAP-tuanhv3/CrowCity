### Manual Studio verification protocol (recommended for next playtest)

When opening the place in Studio for ad-hoc verification, the following Server Console (F9 → Server tab → Command Bar) probe confirms all Sprint 2 contracts in ~3 seconds. Paste verbatim:

```luau
local SS = game:GetService("ServerStorage")
local TickOrch = require(SS.Source.TickOrchestrator)
local CSM = require(SS.Source.CrowdStateServer)
local MSM = require(SS.Source.MatchStateServer)
local RL = require(SS.Source.RoundLifecycle)

local before = TickOrch.getCurrentTick()
task.wait(2)
local after = TickOrch.getCurrentTick()
print(string.format("[1] TickOrch advanced %d ticks in 2s (expect ~30 at 15Hz)", after - before))

print(string.format("[2] MSM state: %s (expect Lobby)", MSM.get()))

CSM._setTestFanoutInterceptor(function() end)
CSM.create("999", {
    crowdId = "999", position = Vector3.new(),
    radiusMultiplier = 1.0, radius = 5.0, count = 10, hue = 1,
    activeRelics = {}, state = "Active", tick = 0,
    stillOverlapping = false, timer_start = nil,
})
print(string.format("[3] CSM create: get('999') = %s", CSM.get("999") and "OK" or "nil"))
CSM.updateCount("999", 5, "Absorb")
print(string.format("[4] CSM updateCount +5: count = %d (expect 15)", CSM.get("999").count))
CSM.updateCount("999", 999, "Absorb")
print(string.format("[5] CSM clamp ceiling: count = %d (expect 300)", CSM.get("999").count))
CSM.destroy("999")
print(string.format("[6] CSM destroy: get('999') = %s (expect nil)", tostring(CSM.get("999"))))
CSM._setTestFanoutInterceptor(nil)

RL._setCSMOverride({
    create = function(id, _) print("    [RL→CSM] create " .. id) end,
    destroy = function(id) print("    [RL→CSM] destroy " .. id) end,
})
RL.createAll({})
print(string.format("[7] RL createAll empty: Janitor = %s", tostring(RL._getJanitor() ~= nil)))
RL.destroyAll()
print(string.format("[8] RL destroyAll: Janitor = %s (expect nil)", tostring(RL._getJanitor())))
RL._setCSMOverride(nil)

print("=== Sprint 2 manual verification COMPLETE ===")
```

## Advisory Notes / Tech Debt

1. **QA plan not run** — `/qa-plan sprint` was warned in `sprint-2.md` header but never executed. Test coverage is comprehensive but formal QA test cases (manual walkthroughs, edge-case bug-hunt scope, sign-off matrix) not pre-defined. **Recommend** running `/qa-plan sprint` before Sprint 3, OR running `/team-qa sprint` directly to generate test cases + sign-off in one pass before stage advancement.

2. **AFKToggle wiring deferred** — MSM 2-8 deliberately did NOT wire `Network.connectEvent(AFKToggle, ...)` despite the original story spec listing it as a "stub". Reasons: (a) Network.connectEvent requires `Network.startServer()` to have created the RemoteEvent instance, which fails in headless TestEZ context; (b) ADR-0010 §4-Check Guard validation must wrap the handler before it accepts traffic — wiring without validation creates a security gap during the story-001 → 007 window. The `_afkToggleConnection` field + `_getAFKToggleConnection` getter remain as scaffolding; story-007 of MSM wires the validated handler. Inline `TODO(story-007)` documents the swap location. **No production impact** — AFKToggle traffic is silently ignored until story-007 lands.

3. **Selene 7 advisory warnings** — all in template-original code (CustomAnalytics unused service / Stage1 + Stage2 FtueStage unused player param / findFirstChildWithAttribute unused index param). No action; not introduced by Sprint 2.

4. **CI enforcement gap** — TestEZ headless on GitHub-hosted ubuntu-latest is warn-only because Roblox Studio is not available on Linux. Lint + audit gates are blocking. To make TestEZ blocking in CI, configure either a self-hosted macOS/Windows runner with Studio installed OR a `macos-latest` GitHub-hosted runner (incurs ~10 min Studio bootstrap per CI run).
